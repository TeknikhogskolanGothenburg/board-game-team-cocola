# User stories

Use this template when writing the user stories:

"As a < type of user >, I want < some goal > so that < some reason >."

Create a new Markdown file for each userstory and create a toc.yml in the userstories folder, and add each userstory to the TOC (table of content).

Read more on userstories at [https://www.mountaingoatsoftware.com/agile/user-stories
](https://www.mountaingoatsoftware.com/agile/user-stories
)

# Our userstory
* Som utvecklare vill jag skapa ett login system för att skapa spel och spelare.
* Som utvecklare vill jag skapa en databas som sparar skapade spel och spelare för att spelare ska kunna returneras till sina spel.
* Som utvecklare vill jag skapa funktioner för att programmet ska fungera säkert.
* Som utvecklare vill jag skapa flera views för att spelare ska kunna logga in eller fylla i nya inloggningsuppgifter för att börja spela.
* Som utvecklare vill jag skapa ett black jack spel för att det är roligt och relativt enkelt att göra.
* Som spelare vill jag ha andra att spela spelet med för att det är roligt.
* Som spelare vill jag att mitt spel sparas om jag eller motspelaren inte hinner spela färdigt, för att kunna återuppta speler.
* Som utvecklare vill jag kunna spara påbörjade spel för att spelare ska kunna återgå till påbörjade spel.
* Som spelare vill jag att ingen annan ska kunna komma till mitt spel och påverka resultaten.
* Som utvecklare vill jag att endast skaparen av nytt spel ska kunna återuppta spelet, för att inte obehöriga ska kunna spela ett påbörjat spel.
* Som spelare vill jag komma till siffran 21 eller nära 21 för att vinna spelet.
* Som spelare vill jag att mina kort väljs slumpmässigt för att spelet ska vara rättvist.
